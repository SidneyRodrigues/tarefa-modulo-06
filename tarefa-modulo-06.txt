Programação Orientada a Objetos (OO)

A programação orientada a objetos é um paradigma que organiza o software em objetos, que são 
instâncias de classes. As classes definem as propriedades e métodos que os objetos podem ter.

Características principais:
  Encapsulamento: Os dados e métodos que operam sobre esses dados são encapsulados dentro de objetos.
  Herança: Classes podem herdar propriedades e métodos de outras classes, promovendo a reutilização de código.
  Polimorfismo: Permite que uma função ou um método tenha diferentes implementações dependendo do contexto.
  Abstração: Permite esconder detalhes complexos e mostrar apenas as funcionalidades essenciais.


Programação Procedural

A programação procedural é um paradigma de programação que se baseia no conceito de procedimento
ou rotina, também conhecido como função. Este paradigma foca em uma sequência de instruções que
 dizem ao computador o que fazer, passo a passo.

Características principais:
  Sequência de Comandos: O programa é composto por uma sequência de instruções que são executadas uma após a outra.
  Uso de Funções: As tarefas são divididas em funções ou procedimentos, 
  que podem ser reutilizados em diferentes partes do programa.
  Variáveis Globais e Locais: Dados podem ser armazenados em variáveis globais 
  (acessíveis por todo o programa) ou locais (acessíveis apenas dentro de uma função).
